# GitHub Pages Browser AI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the approved GitHub Pages browser-side AI architecture so the deployed static app no longer depends on `/api/simulator/*`.

**Architecture:** Keep the React UI flow intact, but replace endpoint calls with browser-safe service functions. Reuse existing DeepSeek transport, response normalization, event seed, answer formatting, and retry helpers. Keep `server.ts` as optional legacy/local backend code while removing it from the Pages production build.

**Tech Stack:** React 19, Vite, TypeScript, DeepSeek Chat Completions API, GitHub Actions Pages deployment, `node:assert/strict` + `tsx` tests.

---

## File Structure

- Create `src/services/ai/errors.ts`: typed AI/service error class and UI-facing error codes.
- Create `src/services/ai/env.ts`: reads `VITE_DEEPSEEK_API_KEY`, `VITE_DEEPSEEK_BASE_URL`, and `VITE_DEEPSEEK_MODEL` from Vite env.
- Create `src/services/ai/deepseekBrowserClient.ts`: browser-facing wrapper around DeepSeek JSON calls with typed error conversion.
- Create `src/services/simulation/prompts.ts`: browser-safe prompt builders extracted from `server.ts`.
- Create `src/services/simulation/simulationService.ts`: endpoint-equivalent browser service methods.
- Modify `src/App.tsx`: replace every `/api/simulator/*` fetch with `simulationService` method calls.
- Modify `package.json`: make `dev` run Vite and `build` run frontend build only; keep optional server scripts.
- Modify `.github/workflows/deploy-pages.yml`: deploy the `dist` artifact generated by CI, without publishing a `gh-pages` branch.
- Modify `.env.example`: document `VITE_DEEPSEEK_*` variables.
- Add tests next to new modules using the existing `*.test.ts` style.
